Best practice
Using classes vs. ids

30 June 2020

Id and class attributes are a way of defining your selectors/tags which are simply notations we can assign to elements so that we can style them in a unique way. It is a great way to easily style the same HTML element in different ways. The function of id’s and classes overlap significantly although there are some key differences between them:

Id’s are rare and powerful selectors.
They can only be used once per html element (everything from a start tag to the end tag) and you can’t have more than 1 copy of an id per page. Id’s also take higher priority to class. They are handy when you want to provide and internal link target i.e. to navigate to a specific section of a page from elsewhere on the page.

Classes are the more common and flexible selector.
One class can be used in more than one element and you can assign multiple classes per element, these only need to be separated by a space.

“This is much like ID’s and Classes. Information that is reusable should be kept in a class and information that is totally unique should be kept in an ID.” - Chris Coyier.

So with the definitions and differences between id and class attributes, here are some suggestions for the most helpful practice when it comes to using them:

  1. Take care with how many you are using.
    Keep your css and html files as simple and easy to comprehend as possible. One cool way of minimising the use of class selectors is by remembering that if a div element, has been assigned class selector, e.g ‘container’, and contains other tagged elements, any of these contained elements can be targeted uniquely in the corresponding css file by stating the class selector followed by the target tag. Eg. to target a specific paragraph within class container: .container p { }. This technique reduces the need to create a new class for each element within an already class defined element. Some of the following points may also be able to support us to be clever rather than indulgent with our use of these selectors. Be consistent and considered.

  2. Name them according to their function.
    This will help us, and/or other people looking at these files to follow our logic and make any maintenance as simple as possible.

  3. Keep names short and sweet.
    This keeps things kinder for our eyes and understanding.

    “ Use Names That Are as Short as Possible But as Long as Necessary”- Jens Oliver Meiert

  4. Use ids for elements that appear only once on the page.
    Like a menu, a header and a footer

  5. Use class for elements that appear on the page more than once.
    Like paragraphs, buttons, and links

  6. Remember that an HTML element can have both an id and a class.
    For example - Assign the same class to style elements so they appear identical and then assign a different id to each to control their positions.

  7. Remember elements may be assigned multiple classes at the same time.
    This means that within an elements presentation can be determined by one or more class properties. E.g. If you have a .content class that is applied to all paragraphs and you want to add a border to one certain paragraph, you can simply add another class, like .bordered, to add extra styling.

  8. Be wary when using id’s in collaborative projects.
    Because of the restrictions around id’s there is a higher risk of contradictions and confusion when other people are working on the same document as you. Either clarify your practice around id’s and classes with collaborators or stick to using classes to achieve your desired styling.

This is not an exhaustive list, but hopefully a thought provoking one. Understanding the rules, establishing your own practice and clarifying practices amongst collaborators seems like a great start to a healthy relationship with id and class selectors. Their proper application relates to function and also ease of understanding. As with anything, clear communication is a great foundation to build upon.
